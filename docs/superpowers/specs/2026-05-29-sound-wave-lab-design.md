# 声音波形实验室设计说明

## 目标

新增一个独立的“声音波形实验室”界面，把当前详情弹窗中的交互式正弦波图解拆出来，避免弹窗空间不足、图解被遮挡或裁剪的问题。该界面用于科普声音基础概念，重点解释频率、振幅、相位、波长与听感之间的关系。

## 范围

本阶段实现独立实验界面，不加入麦克风录音。录音功能依赖浏览器权限与 HTTPS 环境，后续可作为第二阶段扩展。

## 入口

在“什么是声音”详情弹窗中的图解区域提供“打开声音波形实验室”入口。点击后关闭详情弹窗，并把当前页面切换到实验室界面。实验室顶部提供“返回知识库”按钮，返回后保留原有知识库首页体验。

## 功能

实验室界面包含以下功能：

- 波形类型切换：正弦波、方波、三角波。
- 参数控制：频率、振幅、相位三个滑块。
- 声音播放：播放当前参数对应的示例音，支持停止播放。
- 波形可视化：用大尺寸 SVG 展示当前波形，并标注振幅、周期/波长、频率与相位。
- 实时说明：根据当前参数显示频率、振幅、相位变化对听感和图形的影响。
- 概念卡片：展示频率、振幅、相位、波长的简明解释。

## 界面结构

实验室使用单页内部视图切换，不引入路由库。首页仍然是知识库；当用户进入实验室时，主体内容切换为实验室布局。

布局分为三块：

1. 顶部标题区：标题、简短说明、返回按钮。
2. 主实验区：左侧为大尺寸波形图和公式，右侧为波形类型、播放控制、滑块和实时说明。
3. 概念区：用四个概念卡片解释频率、振幅、相位、波长。

## 视觉方向

保留当前网站的深色科技风格和绿色/橙色强调色。实验室不做营销页，不使用大面积装饰背景；重点是清晰、稳定、可操作的工具界面。

## 技术实现

使用 React 组件实现，不引入新依赖。声音播放继续使用 Web Audio API。可视化使用 SVG 生成路径，避免 canvas 在测试和 SSR 环境中的复杂性。

新增组件建议：

- `src/components/SoundWaveLab.tsx`：独立实验室界面。
- `src/components/TopicDetails.tsx`：详情图解区域改为入口卡片，同时保留小型静态概念图。
- `src/App.tsx`：增加 `activeView` 状态，在知识库和实验室之间切换。
